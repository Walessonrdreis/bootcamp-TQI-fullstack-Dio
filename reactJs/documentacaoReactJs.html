Sobre:

O ReactJs é uma biblioteca javascript para criar interfaces de usuário

Criada em 2011 por Jordan Walke no Facebook;

Baseado no XHO, um framework para criação de HTML no PHP

Utilizado a principio no mural de notícias da ferramenta

Depois de 4 anos foi lançado o React NAtive para mobile. E no mesmo ano o UWP(Universal Windows Platform)para desenvolvimento Web

Linguagem declarativa

Baseado em componentes, em resumo pedacinhos que são montados para realizar a aplicação

Sobre o JSX: React não separa as tecnologias colocando marcação em arquivos separados, e sim separa conceitos e mantém o código pouco acoplado chamando-os de componentes.

Sintax Suggar para React.crateElement

O browser nã interpreta o JSX, para isso é necessário um transpilador para que a aplicação possa entender o código. O mais utilizado é o Babel

Renderização

  • Renderização de elementos
  • React DOM

    • O React não trabalha em cima do DOM do navegador, e sim usa o virtual DOM, que controla tudo que está no seu DOM, o React quando o html sofre uma alteração, pega apenas a parte que está diferente e aprica a renderização apenas aquela parte.

    Ou seja, temos toda a árvore, mas só será alterada a parte que foi modificada

  • Nós Raiz

React "pensa" em como a interface deve está a cada momento ao invés de alterá-la com o tempo, o que faz te ganho de performance e evita uma série de erros

Nenhum componente Pai ou filho devem saber se outro componente possui estado ou nãp.

O estado é apenas local ao componente e caso seja necessário enviar algum atributo para outro então é feito via props.
